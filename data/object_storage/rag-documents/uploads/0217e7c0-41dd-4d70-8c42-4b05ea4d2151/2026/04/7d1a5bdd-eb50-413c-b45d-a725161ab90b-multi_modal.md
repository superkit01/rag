# 多模态

---

## 一. 这一章在整个学习路径里的位置

前一章我们默认模型只处理：

- 文本输入
- 文本输出

这一章开始扩展“输入类型”。

多模态不是先让 Agent 变复杂，而是先让模型的感知能力变强。对教学来说，最重要的是先理解：

> 文本模型主要“读文字”，多模态模型既能“读文字”，也能“看图片”。

---

## 二. 什么是多模态

`多模态（Multimodal）` 指模型可以处理不止一种模态的数据，例如：

- 文本
- 图片
- 音频
- 视频

在本仓库里，我们先聚焦最常见、最容易上手的一类：

- 文本 + 图片

对应的常见任务包括：

- 看图理解
- OCR / 表单识别
- 图片问答
- 图文联合推理

---

## 三. 多模态和普通 LLM 的区别

普通文本模型通常是：

```text
文本 -> 模型 -> 文本
```

多模态模型则可能是：

```text
文本 + 图片 -> 模型 -> 文本
文本 + 图片 + 历史消息 -> Agent -> 工具 -> 文本
```

所以，多模态最本质的变化不是流程，而是输入空间。

它解决的问题通常是：

- 用户上传了一张图，模型能不能直接理解
- 模型能不能结合图片和文字一起推理
- Agent 能不能围绕图片内容继续调用工具

---

## 四. 在 LangChain 里怎么传图片

在 `LangChain` 里，多模态输入通常还是通过 `Message` 传给模型，只是 `content` 不再只是字符串，而是内容块列表。

典型写法：

```python
from langchain_core.messages import HumanMessage

message = HumanMessage(
    content=[
        {"type": "text", "text": "请描述这张图片"},
        {
            "type": "image_url",
            "image_url": {"url": image_data_url},
        },
    ]
)
```

可以把它理解成：

- 文本块告诉模型“要做什么”
- 图片块告诉模型“要看什么”

---

## 五. 图片输入的常见方式

### 1. 远程 URL

```python
{
    "type": "image_url",
    "image_url": {"url": "https://example.com/demo.jpg"},
}
```

优点：

- 简单直接

缺点：

- 依赖外部可访问地址
- 教学示例里不够稳定

### 2. Base64 / data URL

```python
{
    "type": "image_url",
    "image_url": {"url": "data:image/png;base64,......"},
}
```

优点：

- 不依赖外部图片服务
- 更适合本地教学示例和离线材料

这也是本仓库更推荐的讲解方式。

## 六. 本章示例在讲什么

对应代码见：

- [code/multi_modal_agent.py](./code/multi_modal_agent.py)

这一章有两层示例：

### 1. 直接调用多模态模型

先让模型直接看图并回答。

目的：

- 先确认“模型能不能看懂图片”
- 不把学习重点过早转移到 Agent 编排

### 2. 把多模态模型放进 Agent

在模型已经能看图的前提下，再让 Agent 处理图文问题。

目的：

- 说明多模态能力可以成为 Agent 的“感知层”
- 为后续 Tool、Runtime、多 Agent 铺路

## 七. 多模态 + Agent 的基本链路

如果把多模态模型放进 Agent，典型流程是：

```text
图片 + 用户问题
-> Agent 理解图像内容
-> 判断是否需要调用工具
-> 输出结果
```

常见场景：

- 用户上传商品图，Agent 查询商品库
- 用户上传报错截图，Agent 查询日志或知识库
- 用户上传票据照片，Agent 调用结构化提取工具

也就是说：

- `多模态模型` 负责感知
- `Tool` 负责执行
- `Agent` 负责决策

## 八. 本章学习时要特别注意什么

### 1. 先学“看懂”，再学“做事”

如果图片内容都还没看懂，就不应该急着引入复杂工具链。

### 2. 多模态不是只等于 OCR

OCR 只是多模态的一种落地方式，多模态更大的价值在于：

- 图文联合理解
- 视觉语义推理
- 把图片内容接入后续 Agent 流程

### 3. 教学示例要优先稳定

相较于依赖公网图片地址，本地图片转 `data URL` 更适合教学仓库。
