# 前端控制台设计

## 1. 定位

前端是一个偏“专家研究台”的控制台，不是简单的聊天窗口。当前已经拆成多页面工作台，而不是把所有操作堆在一个页面里。它同时承担：

- 运营台
- 知识空间入口
- 文档操作入口
- 引用浏览入口
- 评测入口
- 异步任务面板

## 2. 当前主要页面

### Namespace

用于：

- 创建知识空间
- 切换当前知识空间
- 明确当前工作上下文

### Dashboard

展示摘要指标：

- 文档数
- chunk 数
- answer trace 数
- eval run 数

并提供进入各业务页的快捷入口。

### Documents

支持：

- 内联内容导入
- 本地文件上传
- 文档记录列表
- 文档详情页
- chunk / fragment 查看

说明：

- `web` 端上传入口当前只保留“内联内容”和“本地文件上传”。
- `source_path` / `storage_uri` 仍然是后端导入能力，不作为当前前端主入口暴露。

### Chat

支持：

- 问题输入
- 文档过滤
- 流式答案输出
- 带引用答案
- 推荐追问
- 答案反馈

### Tasks

包含两类任务：

- 导入 / 重建索引任务
- 评测任务

支持：

- 查看 `job_kind`
- 查看 `status`
- 查看 `attempt_count`
- 查看 `workflow_id`
- 查看 `error_message`
- 重试任务
- 取消任务
- 进入任务详情页

### Evaluation

支持：

- 单案例评测提交
- run 状态查看
- `attempt_count` 展示
- `workflow_id` 展示
- 重试评测
- 取消评测
- 进入评测详情页

### History

用于查看历史 answer trace，便于复盘和继续追问。

## 3. 详情页

### 文档详情

支持：

- 查看 `source_uri`
- 查看 `storage_uri`
- 查看 chunk 列表
- 查看 fragment 原文

### 任务详情

支持：

- 查看导入任务 `source_uri`
- 查看导入任务关联文档
- 查看评测任务结果明细
- 查看 `workflow_id`
- 查看错误信息

## 4. 轮询机制

当前前端会在以下条件成立时自动轮询：

- 存在 `pending`
- 存在 `running`
- 存在 `cancelling`

轮询目标：

- `/sources/jobs`
- `/eval/runs`
- 以及关联的文档、trace、summary 数据

轮询间隔当前为 `2s`，适合演示和 MVP 阶段。

## 5. 包管理与构建约束

- 使用 `pnpm`
- `packageManager` 固定为 `pnpm@10.12.1`
- `web/.npmrc` 使用 `https://registry.npmmirror.com/`

这样做的原因：

- 避免 npm/pnpm 混用
- 提升国内网络环境下的安装稳定性
- 保持锁文件和 CI 行为一致

## 6. 设计上的边界

前端只负责：

- 消费 API
- 展示状态
- 驱动任务操作

前端不负责：

- 自行判定导入完成
- 自行计算引用可信度
- 自行拼装业务规则

这些规则都必须由后端统一给出。

## 7. 后续可扩展方向

- 文档过滤状态跨页面持久化
- 任务筛选与搜索
- 文档树视图
- 引用片段高亮
- 多案例评测编辑器
- 评测历史对比
- 成本和延迟看板
- ACL 提示与来源标签
