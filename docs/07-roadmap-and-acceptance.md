# 路线图与验收标准

## 1. Phase 划分

### Phase 0：平台底座

目标：

- 仓库骨架
- 数据模型
- 文档解析/切块/索引
- 基础问答 API
- 最小前端界面
- 本地部署脚本

当前状态：

- 已基本完成

### Phase 1：研究问答 MVP

目标：

- 知识空间管理
- 引用跳转
- 问答历史
- 反馈闭环
- 基准评测
- 任务治理

当前状态：

- 大部分已完成
- 还缺正式评测集、运维观测、缓存/限流落地

### Phase 2：企业化增强

目标：

- 企业 SSO
- 文档级 ACL
- 协同平台连接器
- 审计日志
- OpenTelemetry
- 成本和延迟看板
- 灰度/回滚策略

当前状态：

- 未开始，只有数据模型扩展位已预留

### Phase 3：平台化扩展

目标：

- 报告生成
- 专题集合
- 定时同步
- 多模型路由
- 私有模型适配
- 人工审核流

当前状态：

- 未开始

## 2. 验收指标

首版验收目标沿用最初规划：

- 离线 gold set `Recall@10 >= 0.80`
- 引用精度 `>= 0.90`
- groundedness `>= 0.85`
- 问答 `p95 <= 10s`
- 单批 1,000 份文档导入成功率 `>= 99%`
